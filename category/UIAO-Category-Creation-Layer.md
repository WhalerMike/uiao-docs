# UIAO Category-Creation Layer

## Overview

The UIAO Category-Creation Layer is the strategic framework that defines UIAO as the creator and owner of a new market category: the **Compliance Control Plane (CCP)**. This layer stops UIAO from being "a product in a market" and makes UIAO the market itself.

---

## 2. Category Definition (Formal + Analyst-Ready)

A **Compliance Control Plane** is a deterministic system that ingests configuration evidence, normalizes it into a machine-readable representation, evaluates it against formal control logic, detects drift, enforces policy where allowed, and emits auditor-ready OSCAL artifacts with full provenance and POA&M linkage.

This definition is:
- Precise
- Technical
- Analyst-friendly
- Hard for competitors to fake

---

## 3. Category Boundaries (What's In / What's Out)

### 3.1 What's IN the Category

A system must have ALL of the following to be a Compliance Control Plane:

- Deterministic evidence ingestion
- Normalized IR model
- Formal control logic (KSI-like)
- Drift detection
- POA&M automation
- Enforcement adapters
- OSCAL-native outputs
- Provenance manifests
- Evidence graph
- Compliance data lake

If a product lacks even one of these, it is not a Compliance Control Plane.

### 3.2 What's OUT of the Category

These are adjacent categories, not competitors:
- SSPM
- CSPM
- GRC
- SIEM
- SOAR
- Ticketing systems
- Documentation generators
- SCuBA wrappers
- OSCAL exporters

This boundary is what keeps the category pure.

## 5. Category Maturity Model (Stages 0–4)

### Stage 0 — Manual Compliance
- Evidence in spreadsheets
- Controls interpreted manually
- Drift invisible
- ATO takes months

### Stage 1 — Scanning
- SSPM/CSPM tools
- Snapshot-based
- No OSCAL
- No enforcement

### Stage 2 — Workflow
- GRC tools
- Ticketing
- Document management
- Still manual evidence

### Stage 3 — Automation
- Some evidence ingestion
- Some normalization
- Some drift detection
- No enforcement
- No OSCAL

### Stage 4 — Compliance Control Plane (UIAO)
- Deterministic evidence
- Formal control logic
- Drift → POA&M
- Enforcement adapters
- OSCAL-native
- Provenance-backed
- Zero-trust aligned

**UIAO is the only Stage 4 system.**
